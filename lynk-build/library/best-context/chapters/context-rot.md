---
name: Context rot
description: Answer quality drops as context grows — even relevant context — and it starts dropping before the window is full. Read when answers degrade as a session or prompt grows, or someone proposes "just add more context."
---

This is the core problem every other chapter in this book exists to fight: answer quality drops as more context is added — *even context that looks relevant* — and the drop starts well before the window is full. The name and the measurement are sourced: Chroma Research's *Context Rot: How Increasing Input Tokens Impacts LLM Performance* (2025) tested 18 models and found performance grows increasingly unreliable and non-uniform as input length rises, even on trivially simple retrieval tasks. It builds with every extra token added to context, relevant or not; it has no single trigger — it's continuous, as noise-to-signal rises; and the result is that attention dilutes across more tokens and the answer measurably gets worse.

The trap hiding inside it: "more context means more accuracy" is *also* true — for curated, scoped context that a task actually needs. Rot is about raw volume. The resolution is that context should *compound* (more of the right things, loaded when applicable), never *pile* (everything, always) — and the difference between compounding and piling is everything else in this book.

Rot has a second axis: placement. The same Chroma report found that *how* information is presented — where it sits, what surrounds it — moves outcomes even when the information is present: models don't use their context uniformly. Volume decides how much attention exists; placement decides where it lands. Even a single distractor measurably cuts retrieval, and the effect grows with length (Chroma, 2025). The onset is earlier than the advertised window suggests, and the cause is architectural, not capacity: degradation shows by ~32K tokens on hard-retrieval benchmarks, driven by how attention favors early tokens and leans on surface string-matching (see [why-context-rots](why-context-rots.md) for the thresholds and mechanisms in full).

**More context is never free: ask "does this token help," not "is there room."** Concretely (constructed, illustrative): same model, same question — 500 tokens containing just the relevant doc answers correctly; 50,000 tokens with that same doc buried in meeting notes answers measurably worse, sometimes wrong. Nothing was removed; the answer degraded anyway. And even well-intentioned context rots: *Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?* (ETH Zurich, 2026; 138 repositories, 5,694 pull requests) found LLM-generated context files cut task success ~3% while adding over 20% inference cost, while human-written ones gained ~4% and only when limited to non-inferable details. Adding context is an intervention with side effects, not a free vitamin.

Rot is the reason knowledge belongs in small, routed pieces loaded on demand rather than one big prompt: you route by cheap pointers and pull only the pieces a task actually needs. Every other chapter here is a way to compound context without piling it. See [four-operations](four-operations.md) for the levers and [progressive-disclosure](progressive-disclosure.md) for the economy that keeps growth from becoming rot.
