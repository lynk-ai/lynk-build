Progressive disclosure is loading context lazily: a small index of everything is always present, and the full body of a page loads only when a task actually reaches for it. The point is the economics — cost scales with what's *used*, not what *exists* — so the price of a session tracks what the task touches, not what the library contains, and a library can grow without making every session more expensive.

It works as two prices, three stages, and one catch. Two prices, paid at different times: the pointer price (a name plus one-line description per page) is paid always, up front, for everything that exists; the full price (the body of a page) is paid only when a task matches and opens it. (Derived: cost = discovery-cost + Σ full-price over touched pages, so adding a hundred untouched pages only grows the small first term.) Three stages charge those prices, codified by Anthropic's Agent Skills and labelled L1/L2/L3 in community write-ups: discovery loads a name and one-liner for everything (~100 tokens each), always paid; activation loads the full page body on match; execution loads linked references and detail on touch. Inside execution there are two cost shapes — detail *read* into context pays the full token price of the text, while work *executed* outside it (a script through a shell) pays only for the printed output, so the cheapest disclosure is content that never enters the window at all. And the catch (our call): an agent that reads one page at a time is running a *habit* — nothing stops it from reading the whole folder, and the moment it does the economics collapse; a *contract* is an enforced protocol where the agent routes through an index it must trust and something keeps that index honest (a change that touches a page but not its index is rejected). You can't build a product on a habit.

**Pay a pointer for everything that exists; pay full price only for what the task actually touches.** Anthropic's Agent Skills is the real case: a skill exposes its name and description always, its `SKILL.md` body loads only when the task matches, and its bundled files load only when the procedure reaches them — the pattern spread fast on this shape, roughly 40 clients within ~90 days of the spec opening (Dec 2025) and hundreds of thousands of skills across marketplaces (reported, 2025–26).

Any library built this way inherits the mechanism: an index is the discovery layer, page bodies are activation, and linked detail is execution — and where that detail is executable work rather than text, running it outside the window so only its output returns is the cheapest disclosure of all.
