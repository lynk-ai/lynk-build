name: Compression
description: Compression is lossy by design — what survives (conclusions, citations, thresholds) is a decision, not an accident; the good escape shape is a condensed, cited return. Read when a long session, tool result, or subagent return must shrink — deciding what survives the summary and when to trigger it.
---

This is the COMPRESS lever of [four-operations](four-operations.md) as a principle: shrinking context is *lossy by design*, so the real decision is not "should we summarize" but *what is allowed to survive*. A compression that keeps the wrong things (narrative, dead ends) and drops the right ones (conclusions, citations, thresholds) is worse than none — it launders noise into something that looks authoritative.

So the decision has four parts. What survives: conclusions, decisions with their why, citations and pointers back to full sources, thresholds and numbers. What dies: raw logs, exploration dead ends, repetition — anything a kept pointer can re-fetch. When: a decision, not a token threshold — compact at boundaries (sub-task resolved, trajectory converged), not at an arbitrary size (see [memory-shapes](memory-shapes.md) for the sourced result that rubric-triggered compaction beats threshold triggers at a fraction of the cost). And the escape shape: a condensed, cited return, where every surviving claim keeps a pointer to where the full version lives, so compression is reversible by re-fetch.

Prune before you summarize — the cheapest lever first. In Anthropic's research-agent benchmark, tool results were 96.3% of a 335K-token context, so mechanical tool-result clearing (rule-based deletion, no inference cost) bounded peak context to 173K with the same work done, while lossy compaction roughly halved peak (335K → 169K) at the cost of extra turns, its summary only ~2.8K tokens (source: Anthropic Claude Cookbook, *Context-Engineering Tools*, 2026). The order matters: delete redundant tool output first (free), summarize only what survives (paid, lossy). One caveat with teeth — clearing invalidates the KV-cache prefix, so each clear pays a cache-rewrite cost, and Anthropic's `clear_at_least` exists to ensure a clear frees enough to be worth it (source: Anthropic context-editing docs, 2026).

**Compression is a decision about what survives, made at a boundary — and every survivor keeps a pointer home, so what was dropped can always be re-fetched.** The general shape: a research subagent returns a short summary with links back to its sources rather than the raw pages it read; a conversation compaction keeps the decisions and open threads and drops the transcript. In both cases every survivor is a re-fetchable pointer, so nothing is truly lost. See [four-operations](four-operations.md) for the lever this chapter deepens, and [pointers-not-content](pointers-not-content.md) — the same discipline applied to the discovery layer instead of the return path.
