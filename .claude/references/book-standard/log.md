## [2026-07-21] New rule: example-domain-fit — examples match the book's declared scope (both modes)
Added chapter `example-domain-fit.md` (rule `quality/example-domain-fit`, warn): an illustrative example's domain must fit the book's declared scope — a domain-scoped book draws examples from its domain; a general/method book uses domain-neutral (still concrete) examples; real cited examples that ARE the subject count as in-domain. Supports both modes deliberately (some books/chapters are domain-bound, some general), decided when the book is created. Wired into the build path: `write-a-book` prerequisite now notes that declaring the audience also sets the example mode, and `write-a-section` step 4 (Example) now says to match the example's domain to the book's scope. Index gained a Part II row (8e) + a red-flag row. Companion to write-for-the-shelf (which governs referent resolution) — this governs example domain. Applied identically to both book-standard copies.
