# Index

**Book 5 — Subagents.** How to split work across more than one agent: how control passes between them, how their context is walled off, what you hand a blind worker, and the three multi-agent shapes worth knowing (fan-out for breadth, gated pipelines, panels for judgment). This book is the full home of the ISOLATE-lever material that began as Book 1's `isolation-and-strict-briefs` page — **that page graduated here on 2026-07-07** (stub + pointer left behind). It cites Book 1 for the principles and Book 3 for the return-budget economics; it never restates them.

## Pages

| Page | Type | Description |
|---|---|---|
| `call-return-vs-handoff` | principle | Call/return keeps the parent driving; a handoff transfers control one-way — default to call/return, reserve handoffs for work you'll never steer again. |
| `gates-and-ablation` | principle | Fixed roles publish structured artifacts checked at every handoff, and discrete stages let you ablate one piece to measure its worth — a delta, not an opinion. |
| `isolation-is-a-decision` | principle | Whether a subagent shares or isolates context is a design choice with a real cost — frameworks default differently, so decide it on purpose. |
| `orchestrator-workers` | principle | A lead agent decomposes a task and fans it out to parallel isolated workers that return per a strict contract (classically condensed findings; in our pointer flow, pointers with a hook delivering the chapters) — a pattern that costs ~15x the tokens and must earn it. |
| `persona-panels` | principle | Several agents answer independently then see each other's answers; agreement is cheap confidence and disagreement flags a human decision — but the diversity must be engineered. |
| `the-strict-brief` | principle | A blind worker can only act on what you hand it — the four-part brief (objective, output format, tool guidance, boundaries) and a strict, pre-declared return contract (never a raw transcript) decide everything. |
