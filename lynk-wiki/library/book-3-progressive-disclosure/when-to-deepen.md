# When to add a disclosure level

**What it is** — the judgment you face when a home outgrows itself: split *sideways* (more pages at the same level) or *deepen* (add a new discovery layer — an `index.md` of its own, so a page becomes a book). Both are progressive disclosure; they differ in whether you add breadth or depth. This page is the decision that routes between the two mechanical rules that carry the moves out — it does not restate them.

**Mechanics** — the decision:

| Move | When | Mechanism (gate-enforced) |
|---|---|---|
| **Split sideways** | The home is still *one concept*, just too long or buried under caveats. | the Book Standard · `page-budget` — the short current page keeps the name; specifics move to sibling pointer pages. |
| **Deepen a level** | A sub-part has become *several concepts* a reader cites independently — it needs its own discovery layer. | the Book Standard · `graduation` — the sub-part earns its own `index.md` and becomes a book; the old home shrinks to a stub. |

The tell for deepening: **you find yourself wanting an index for the sub-part.** An index is a discovery layer ([three-stages](three-stages.md)), and you only need one when there are several things to disclose. One overgrown concept wants sibling pages; a *cluster* of independently-cited concepts wants an index — and an index is a new level. Don't deepen a single concept (an index over one page is pure overhead), and don't flatten a cluster (a pile of sibling pages with no index is an un-navigable shelf).

**Takeaway** — **deepen a level only when a sub-part has grown its own cluster of independently-cited concepts that need their own index; short of that, split sideways.**

**Example** — this very book. `progressive-disclosure` began as one Book 1 page. It didn't merely get long — it grew five concepts (the economics, the three stages, habit-vs-contract, pointers-not-content, convergent-evolution) that other pages cite independently. Five concepts need a discovery layer, so the page earned its own `index.md` and deepened into a book (per graduation). A page that had only gotten long would have split sideways into sibling pages instead.

**In this system** — the two moves are gate-enforced mechanics (`page-budget` warns on oversize; `graduation` rejects an unstubbed page→book move); this page is the *judgment* that decides which one applies, so a writer isn't left guessing between them. This split is deliberate: the *whether/which* is a principle (it lives here), the *how* is a rule (it lives in the standard, where the gate can check it). → See the Best Context book · `living-sources` for why the refactor must happen at all, and [pointers-not-content](pointers-not-content.md) for what the new index layer must (and must not) carry.
