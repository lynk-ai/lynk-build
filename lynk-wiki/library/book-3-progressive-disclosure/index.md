# Index

**Book 3 — Progressive Disclosure.** The why and how of lazy context loading: a small index of everything is always present; the full body of a page loads only when a task reaches for it. So a library can grow without making every session more expensive. This book graduated from Book 1's `progressive-disclosure` page (2026-07-07), which is now a stub pointing here.

**Page shape:** every page follows the concept template (the Book Standard · `page-template`): *What it is → Mechanics → Takeaway → Example → In this system.*

## Part I — The wager

| Page | One-liner |
|---|---|
| `the-economics.md` | Pay a pointer for everything that exists; pay full price only for what the task touches — cost scales with use, not size. |
| `three-stages.md` | Discovery → activation → execution (the community's L1/L2/L3): the mechanism, codified by Anthropic's Agent Skills and general to any index-plus-pages library. |
| `read-vs-execute.md` | Inside execution, two cost shapes: detail read into context (pay the full token price) vs code executed outside it (only the output is paid) — the cheapest disclosure never enters the window. |
